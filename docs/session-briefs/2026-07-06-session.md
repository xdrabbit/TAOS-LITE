## 2. The prediction model (architecture the next session MUST understand)

- **Unified n-gram model**: unigram + bi/trigram + phrase, recency-weighted with **exponential decay, 30-day half-life** (`weight = 0.5^(ageDays/30)`).
- Trained **PER DIRECTION** from the `taos_lite_translations` history table, columns `original_text` + `created_at`.
- **⚠️ CRITICAL DATA QUIRK:** the `source_lang` column is **UNRELIABLE / effectively INVERTED** (en-tagged rows are mostly Spanish, ~640 vs 279). The model builder therefore **DETECTS language from `original_text` itself and ignores `source_lang`**. **Any future work touching this history MUST NOT trust `source_lang`.** Also ~313 leaked prompt/system-text junk rows are filtered by pattern.
- **Storage:** model lives in Supabase table `taos_lite_predict_models` (**NOT files** — Vercel runtime FS is read-only). Client reads via a cached server route. `public/models/*.json` are dev-inspection artifacts only (gitignored).
- **Nightly rebuild:** Vercel Cron `GET /api/predict/rebuild` at **08:00 UTC**, authed via `CRON_SECRET` (now set in Vercel, and rotated). **Manual rebuild:** "rebuild suggestions" button (authed POST) or `node scripts/build-predict-model.mjs` locally with `SUPABASE_SERVICE_ROLE_KEY`.

## 3. KEY INSIGHT to preserve (the "wormhole")

The translations history is **AI-PARAPHRASED output, not raw human speech.** So the model learns the AI's *regularized* phrasing of Tom+Liz's meaning — which is **lower-entropy / more predictable** than real human text, giving the autocomplete "warp speed" (a model predicting a model). **Tradeoff:** it types fast + correct, but not authentically "like Tom/Liz."
